# LogParser

Проект **LogParser** предназначен для парсинга лог-файлов и анализа данных о поисках и открытиях документов. Приложение выполняет два основных задания:

1. **Задание 1**: Подсчёт количества поисков документа `ACC_45616` через карточку поиска.
2. **Задание 2**: Подсчёт количества открытий каждого документа, найденного через быстрый поиск, за каждый день.

## Структура проекта

```
logparser (корневая директория проекта)
├── input                (папка для входных данных)
├── logs                 (папка для лог-файлов)
├── output               (папка для выходных данных)
├── src
│   ├── main
│   │   ├── java
│   │   │   └── org.company
│   │   │       └── Main.java              (основной Java-класс, если используется)
│   │   ├── resources
│   │   │   ├── log4j2.xml                 (конфигурация логирования)
│   │   │   └── MainApp.run.xml            (конфигурационный файл приложения)
│   │   └── scala
│   │       └── com.company.logparser
│   │           ├── model
│   │           │   └── Event.scala                (описание модели событий)
│   │           ├── parser
│   │           │   └── LogParserService.scala     (парсер логов)
│   │           ├── processor
│   │           │   ├── CardSearchAnalyzer.scala   (анализ карточных поисков)
│   │           │   └── QuickSearchAnalyzer.scala  (анализ быстрых поисков)
│   │           └── MainApp.scala                  (основной файл приложения)
├── test                 (папка для тестов)
├── .gitignore           (файл исключений Git)
└── pom.xml              (конфигурация Maven)
```

## Требования

- **Java**: версия **17.0.11** (2024-04-16 LTS)
- **Scala**: версия **3.5.2**
- **Scala code runner**: версия **1.4.3**
- **Apache Spark**: версия **3.5.3**
- **Apache Maven**: версия **3.6.0** или выше

## Сборка проекта

1. **Клонируйте репозиторий**:

   ```bash
   git clone https://github.com/temraq/logparser.git
   ```

2. **Перейдите в директорию проекта**:

   ```bash
   cd logparser
   ```

3. **Убедитесь, что установлены все зависимости**:

   - Java JDK 17
   - Scala 3.5.2
   - Apache Spark 3.5.3
   - Apache Maven 3.6.0 или выше

4. **Соберите проект с помощью Maven**:

   ```bash
   mvn clean install
   ```

## Запуск приложения

После успешной сборки запустите приложение следующей командой:

```bash
mvn exec:java -Dexec.mainClass="com.company.logparser.MainApp"
```

**Примечание:** Убедитесь, что у вас правильно настроена конфигурация `MainApp.scala`. В проекте используется конфигурационный файл `MainApp.run.xml`, который находится в директории `src/main/resources`. Этот файл содержит специфические настройки приложения, необходимые для его корректной работы.

## Конфигурация `MainApp`

- **Конфигурационный файл**: `src/main/resources/MainApp.run.xml`

Убедитесь, что файл `MainApp.run.xml` содержит необходимые настройки для вашего окружения, такие как пути к входным и выходным данным, параметры запуска и другие конфигурации.

## Результаты выполнения

### Задание 1

- **Количество поисков документа `ACC_45616` через карточку поиска**: **347**

Результат выводится в консоль при запуске приложения.

### Задание 2

- **Количество открытий каждого документа, найденного через быстрый поиск, за каждый день**

Результаты сохраняются в CSV-файле по пути:

```
output/doc_open_counts/
```

Каждый CSV-файл внутри этой директории содержит данные с колонками:

- **date**: дата в формате `dd.MM.yyyy`
- **docId**: идентификатор документа
- **count**: количество открытий

**Примечание:** Для просмотра результатов перейдите в директорию:

```
корневая директория проекта\output\doc_open_counts\
```

Откройте CSV-файлы в любом табличном редакторе (например, Microsoft Excel).
